# Claude Code プロジェクトガイドライン

## Just コマンドの使用

**重要**: 開発タスクの実行には`just`コマンドを使用すること。

### 基本ルール

* `justfile`に定義されたタスクが存在する場合は、必ずそれを使用すること
* 直接的なbashコマンド実行（`cd backend && uv run ...`）は避けること

### 主要コマンド

@justfile を確認すること

## Git Commit ポリシー

**重要**: すべての作業が完了した際には、必ずgit commitを実行してください。

### Commitルール

1. タスクが完了したら、変更内容を確認するため`git status`と`git diff`を実行する
2. 適切なファイルを`git add`でステージングする
3. 変更内容を説明する明確なコミットメッセージでcommitを作成する
4. コミットメッセージには以下を含める：
   - 変更の概要（1行目）
   - 必要に応じて詳細な説明


## ブランチ命名規則

**重要**: ブランチ名は以下の形式に統一すること。

### 命名ルール

1. 形式: `<type>/<issue-number>-<short-description>`
2. `<type>` は `feat` / `fix` / `chore` / `docs` / `ref` / `test` のいずれか
3. `<issue-number>` は GitHub Issue 番号（必須）
4. `<short-description>` は英小文字のkebab-case（スペース禁止、英数字とハイフンのみ）

## Pull Request 作成ポリシー

**重要**: PRを作成する際は、必ず`.github/pull_request_template.md`のテンプレートに従って記載すること。

### PRルール

1. PRを作成する前に、変更内容を確認する
2. テンプレートの全項目を埋める
3. チェックリストを確認して適切にチェックする
4. 動作確認結果を具体的に記載する

## Issue/PR 取得ポリシー

**重要**: issueやPRの取得には、ghコマンド(GitHub CLI)を活用すること。

## ドキュメント記述規約

### 技術用語の扱い

**重要**: 技術用語を不用意に日本語変換しないこと。

* 技術用語は原則として英語のまま使用すること
* 日本語訳が分かりにくい場合は、英語表記を優先すること

### 句読点の使用

**重要**: 日本語文章には`.`（ピリオド）ではなく`。`（句点）を使うこと。

* 日本語の文末には必ず`。`を使用すること
* コード中のコメントに`。`や`.`は不要
* 英語の文章では`.`を使用すること
