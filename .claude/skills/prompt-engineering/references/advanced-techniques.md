# 高度なテクニック

## 1. 応答の事前入力（Prefilling）

期待される応答の開始部分を提供して出力形式を制御する。

例: `Assistant: {` で開始させることでJSON出力を強制

## 2. プロンプトの連鎖

複雑なタスクを複数のステップに分割し、各ステップの出力を次の入力として使用する。

例:
- ステップ1: 要件分析
- ステップ2: 設計案作成
- ステップ3: 実装計画

## 3. 長文コンテキストの活用

大量の情報を扱う際のテクニック:
- 重要な情報を最初と最後に配置
- XMLタグで構造化
- 参照すべき情報を明示的に指示

## 4. Few-shot vs Zero-shot

- **Zero-shot**: 例なしで指示のみ（明確な指示で十分な場合）
- **Few-shot**: 2-5個の例を提供（分類、フォーマット変換）
- **Many-shot**: 6個以上の例（複雑な分類）

## 5. Self-Consistency

同じ問題に対して複数の推論パスを生成し、最も一貫性のある答えを選択する。

例: 同じ問題を3つの異なる方法で解かせ、共通する答えを採用

## 6. Tree of Thoughts

複数の思考パスを探索し、各ステップで評価してから次に進む。

例: 意思決定問題で複数の選択肢を並列探索し、各選択肢の帰結を評価

## まとめ

これらの高度なテクニックは、特定の状況で効果を発揮する。タスクの性質に応じて適切に選択すること。
